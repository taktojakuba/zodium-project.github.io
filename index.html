zfetch-rs

zfetch a fast & good looking fetch tool

zfetch — overview
zfetch is a minimal system info fetch tool written in Rust. It displays system information with clean visuals, supports multiple themes, auto-resizes to terminal size, and includes a TUI for easy configuration.

zfetch — installation
Clone the repository and install using: cargo install --path .
Alternatively, use a precompiled binary from the releases section.

zfetch — usage
Run zfetch normally to display system info.
Use flags like --os to change logo, --image to display images, --config to open TUI config, --refresh to update cached values, and --update to refresh config version.

zfetch — configuration
Config is stored at ~/.config/zfetch/config.toml and auto-generated on first run. You can edit it manually or via the built-in TUI.

zrun-rs

A fast, polished TUI shell-script launcher written in Rust

zrun — overview
zrun is a fast TUI-based script launcher. It lets you browse, search, tag, and execute shell scripts with zero external dependencies.

zrun — installation
Install via cargo install --path . or use prebuilt binary.

zrun — usage
Run zrun to open TUI picker.
Use CLI commands like run <name>, list, show, edit, which, and history.
Supports fuzzy search, tagging, and script previews.

zrun — configuration
Config file at ~/.config/zrun/config.toml (not auto-created).
Defines script directories, history limit, and behavior options.

zrun — features
Includes built-in fuzzy search, script tagging via comments, run history, syntax highlighting, and multi-directory priority system for scripts.

zgpu

short desc

zgpu — overview
zgpu is a GPU selection and PRIME offload launcher. It allows running apps on specific GPUs across Intel, AMD, and NVIDIA. Built as a static Rust binary with no runtime dependencies.

zgpu — installation
Add musl target: rustup target add x86_64-unknown-linux-musl
Build: cargo build --release
Install: cargo install --path .

zgpu — usage
Run zgpu alone to see GPU detection summary.
Use options like --use-device to select GPU, --interactive-choose for menu selection, --list-devices to list GPUs, and --verbose for detailed info.
Commands executed after zgpu run on selected GPU.

zgpu — configuration
GPU mappings are saved in ~/.config/zgpu/defined.

zync

short desc

zync — overview
zync is a unified update manager for Fedora Atomic and similar systems. It handles system updates, Flatpak, Homebrew, containers, firmware, and rollbacks from a single command.

zync — installation
Build using cargo build --release (or musl target).
Install binary: sudo install -Dm755 target/release/zync /usr/bin/zync

zync — usage
Run zync with flags like --all to update everything, or specific modules like --flatpak, --rpm-ostree, --podman.
Supports dry-run, non-interactive mode, auto-updates, and rollback operations.

zync — features
Includes rollback support, automatic updates via systemd timers, reboot detection, and cleanup modes.
Works with multiple package managers and tools across different distributions.

zync — logs and exit codes
Logs stored at ~/.local/state/zync.log (or $XDG_STATE_HOME).
Exit codes indicate success, argument errors, or module failures.
